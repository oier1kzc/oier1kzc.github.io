F. Two Subtrees

Statement

给定一个 n (1≤n≤2⋅105)n \ (1 \leq n \leq 2 \cdot 10 ^ 5)n (1≤n≤2⋅105) 个点的有根树，点 111 是根。

给定一个长为 nnn 的数组 a1,⋯ ,ana _ 1, \cdots, a _ na1​,⋯,an​，点 uuu 的权值是 au(1≤au≤2⋅105)a _ u (1 \leq a _ u \leq 2 \cdot 10 ^ 5)au​(1≤au​≤2⋅105)。

给定 m (1≤m≤2⋅105)m \ (1 \leq m \leq 2 \cdot 10 ^ 5)m (1≤m≤2⋅105) 次询问，每次询问给出两个点 x,yx, yx,y，内容如下

  • 考虑在 x,yx, yx,y 子树内的所有点 www 构成的可重集（如果同时在 xxx 和 yyy 的子树中则算两次）。将这些点的权值写下来，找到出现次数最多的数，如果有多个，则找最小的数，为这次询问的答案。

Solution

设一个数据结构 D\mathcal DD 可以支持

  • O(f(n))\mathcal O(f(n))O(f(n)) 的时间内插入或删除一个元素。其中 nnn 是插入元素的总个数。
  • O(g(n))\mathcal O(g(n))O(g(n)) 的时间内计算所有插入元素的众数，若有多个则找到最小的。其中 nnn 是插入元素的总个数。

允许离线。

PreSolution1

考虑设计数据结构 D\mathcal DD，满足 f(n)=O(1)f(n) = \mathcal O(1)f(n)=O(1)，g(n)=O(n)g(n) = \mathcal O(\sqrt n)g(n)=O(n​)。

具体来说，要维护一个可重集，支持 O(1)\mathcal O(1)O(1) 插入一个数，O(n)\mathcal O (\sqrt n)O(n​) 查询最小众数。

其中 nnn 为总共不同数个数。允许离线。

将所有插入的数拿出来，用 (x,c)(x, c)(x,c) 描述数字 xxx 出现了 ccc 次，这样的数对一共有 O(n)\mathcal O(n)O(n) 个。

将所有数拿出来，离线数一遍出现次数，找到这样 O(n)\mathcal O(n)O(n) 个 (x,c)(x, c)(x,c) 数对，以 ccc 为第一关键字、xxx 为第二关键字，按 ccc 从大到小、xxx 从小到大排序。

问题转为，维护一个长为 nnn 的数组，O(1)\mathcal O(1)O(1) 将某个位置赋为 111，O(n)\mathcal O(\sqrt n)O(n​) 查最靠前的数的位置。

取 B=n+O(1)B = \sqrt n + \mathcal O(1)B=n​+O(1)，对每 BBB 个位置分块处理即可。

空间复杂度为 O(n)\mathcal O(n)O(n)。

PreSolution2

对于每个点，求出其子树众数。

对于树上的每个点 uuu，求出其所有儿子中，size\text{size}size 最大的点，记为其重儿子。

设递归函数 dfs 输入为 uuu，计算 uuu 子树的所有点的答案，并返回包含 uuu 子树内所有元素的数据结构 D\mathcal DD。

对于每个点 uuu，先枚举所有轻儿子，递归计算子树答案，并且清空 D\mathcal DD。

再递归重儿子计算子树答案。不清空 D\mathcal DD。

扫一遍所有轻儿子，将 uuu 和轻儿子子树内点的信息加入 D\mathcal DD 中。

此时 D\mathcal DD 中所有元素即为 uuu 子树内所有元素。用 O(g(n))\mathcal O(g(n))O(g(n)) 查答案即可。

时间复杂度为 O(nf(n)log⁡n+ng(n))\mathcal O(n f(n) \log n + n g(n))O(nf(n)logn+ng(n))。

若用 PreSolution1 中所给出的数据结构 D\mathcal DD 维护，时间复杂度为 O(nn)\mathcal O (n \sqrt n)O(nn​)。

MainSolution

对树 dfs，对每个点 uuu，求出

  • fa[u]fa[u]fa[u]，表示节点 uuu 的父亲节点
  • sz[u]sz[u]sz[u]，表示点 uuu 的子树内节点数量。
  • bg[u]bg[u]bg[u]，表示 uuu 再 dfs 序中的位置，下标从 111 开始。

分块，设定阈值 BBB。

对于每个点 uuu，称 uuu 是轻的，当且仅当 sz[u]<Bsz[u] < Bsz[u]<B，否则称其是重的。（由于我不太擅长使用中文，在这里声明以下，下文中的所有“重点”意思均为一个重的点，而非表示强调）

将所有询问离线。

对于每个询问 (x,y)(x, y)(x,y)，xxx 和 yyy 的顺序是无所谓的，不妨设 bg[x]<bg[y]bg[x] < bg[y]bg[x]<bg[y]，以下默认该条件存在。

称一个询问 (x,y)(x, y)(x,y) 是轻的，当且仅当 xxx 是轻的，否则称其是重的。

将询问分成轻重两部分处理。

轻询问

对树做一遍 PreSolution2，遍历到点 yyy 时，处理所有 (x,y)(x, y)(x,y) 的询问。

考虑到 sz[x]<Bsz[x] < Bsz[x]<B，暴力枚举 xxx 子树内所有点，暴力加入 D\mathcal DD 中，然后查询答案。

时间复杂度为 O(nf(n)log⁡n+qBf(n)+qg(n))\mathcal O(n f(n) \log n + q B f(n) + q g (n))O(nf(n)logn+qBf(n)+qg(n))。

若用 PreSolution1 中所给出的数据结构 D\mathcal DD 维护之，时间复杂度为 O(nlog⁡n+qB+qn)\mathcal O(n \log n + q B + q \sqrt n)O(nlogn+qB+qn​)。

重询问

考虑将所有重点 uuu 拿出来，构成点集 SSS。

若 SSS 为空，即只有轻点，即只有轻询问。以下默认 SSS 非空。

观察性质。SSS 是一个连通块。因为若点 uuu 满足 sz[u]≥Bsz[u] \geq Bsz[u]≥B，则 sz[fa[u]]≥Bsz[fa[u]] \geq Bsz[fa[u]]≥B。

于是 SSS 可以看成一棵以 111 为根的树。且这棵树的叶子数量不超过 ⌊nB⌋\left \lfloor \dfrac n B \right \rfloor⌊Bn​⌋，因为每个叶子节点的子树不交，而又都 ≥B\geq B≥B。

将 SSS 剖成若干条链，满足对于每条链 P={p1,⋯ ,pk}⊂SP = \{p _ 1, \cdots, p _ k\} \subset SP={p1​,⋯,pk​}⊂S，有

  • 对于所有 1≤i<k1 \leq i < k1≤i<k，pip _ ipi​ 是 pi+1p _ {i + 1}pi+1​ 的父节点。
  • 对于所有 1≤i<j≤k1 \leq i < j \leq k1≤i<j≤k，有 sz[pi]−sz[pj]<Bsz[p _ i] - sz[p _ j] < Bsz[pi​]−sz[pj​]<B。

把每个点都单独剖成一条链，是合法的。考虑让剖得的链都是极长的，具体地，考虑一个重点 uuu，先递归剖其所有（重点）儿子 vvv 子树，若可以将 uuu 剖到某个 vvv 所在的链 PPP 中（即满足 sz[u]−sz[pk]≤Bsz[u] - sz[p _ k] \leq Bsz[u]−sz[pk​]≤B），则剖到 PPP 中；否则将 uuu 单独剖成一条链 P={u}P = \{u\}P={u}。

接下来说明，这样剖出的链的数量是 O(nB)\mathcal O \left( \dfrac n B \right)O(Bn​) 级别的。考虑链会怎样结束。对于链顶 uuu，

  • 若 u=1u = 1u=1，即是根节点，这样的链只有 111 条；
  • 否则，fa[u]fa[u]fa[u] 存在。若 fa[u]fa[u]fa[u] 的（重点）儿子不仅有 uuu，这可以看成在 fa[u]fa[u]fa[u] 的位置，将多个链合并为了一条。考虑在所有这样的 fa[u]fa[u]fa[u] 的位置算其对 uuu （即链顶数，即链数）的贡献，对于所有 SSS 中的点 vvv，其对 uuu 的贡献有 O(deg(v))\mathcal O(\text{deg}(v))O(deg(v)) 次，其中 deg(v)\text{deg}(v)deg(v) 表示 vvv 的度数，因为 SSS 的叶子数量不超过 ⌊nB⌋\left \lfloor \dfrac n B \right \rfloor⌊Bn​⌋，所以 O(∑deg(v))=O(nB)\mathcal O \left(\sum \text{deg} (v) \right) = \mathcal O \left( \dfrac n B \right)O(∑deg(v))=O(Bn​)。于是这样的链有 O(nB)\mathcal O \left( \dfrac n B \right)O(Bn​) 条。
  • 否则，fa[u]fa[u]fa[u] 的（重点）儿子仅有 uuu 这一个。设包含 uuu 的链 P={p1,⋯ ,pk}P = \{p _ 1, \cdots, p _ k\}P={p1​,⋯,pk​}，其中 pip _ ipi​ 是 pi+1p _ {i + 1}pi+1​ 父节点（p1p _ 1p1​ 即 uuu）。那么将 uuu 和 fa[u]fa[u]fa[u] 剖成两条链，说明 sz[fa[u]]−sz[u]≥Bsz[fa[u]] - sz[u] \geq Bsz[fa[u]]−sz[u]≥B。考虑将所有在 fa[u]fa[u]fa[u] 子树中且不在 pkp _ kpk​ 子树中的点标记，因为 sz[fa[u]]−sz[u]≥Bsz[fa[u]] - sz[u] \geq Bsz[fa[u]]−sz[u]≥B，这样标记的点至少有 BBB 个。并且可以发现，每个点最多被一条链标记。于是这样的链有 O(nB)\mathcal O \left( \dfrac n B \right)O(Bn​) 条。

于是这样剖出来的链的数量是 O(nB)\mathcal O \left( \dfrac n B \right)O(Bn​) 级别的。

对于所有询问 (x,y)(x, y)(x,y)，根据 xxx 在哪条链上，将 (x,y)(x, y)(x,y) 分成 O(nB)\mathcal O \left( \dfrac n B \right)O(Bn​) 类。对于每类询问，分别处理。

对于每条链，设链顶为 uuu，先将 uuu 上所有点加入 D\mathcal DD 中，对整棵树做一遍 PreSolution2。对于分类在该链上的询问 (x,y)(x, y)(x,y)，遍历到点 yyy 时，将 uuu 子树中除了 xxx 子树外的部分删去，查一次答案，再将删去的点加回去。

根据划分出来的链的性质，对于每个询问，额外进行删除和加入点一共是 O(B)\mathcal O(B)O(B) 个。于是这部分总时间复杂度为 O(nB⋅nlog⁡n⋅f(n)+qBf(n)+qg(n))\mathcal O \left(\dfrac n B \cdot n \log n \cdot f(n) + q B f(n) + q g(n) \right)O(Bn​⋅nlogn⋅f(n)+qBf(n)+qg(n))。

若用 PreSolution1 中所给出的数据结构 D\mathcal DD 维护之，时间复杂度为 O(n2log⁡nB+qB+qn)\mathcal O \left( \dfrac {n ^ 2 \log n} B + q B + q \sqrt n \right)O(Bn2logn​+qB+qn​)。

于是处理轻重询问，可以在 O(n2log⁡nB+qB+qn)\mathcal O \left( \dfrac {n ^ 2 \log n} B + q B + q \sqrt n \right)O(Bn2logn​+qB+qn​) 的时间复杂度内完成。

取 B=O(n2log⁡nq)B = \mathcal O \left( \sqrt {\dfrac {n ^ 2 \log n} q} \right)B=O(qn2logn​​) 以平衡之，则上述时间复杂度为 O(nqlog⁡n+qn)\mathcal O \left( n \sqrt {q \log n} + q \sqrt n \right)O(nqlogn​+qn​)，n,qn, qn,q 同阶时为 O(nnlog⁡n)\mathcal O \left( n \sqrt {n \log n} \right)O(nnlogn​)。至此可以解决本题。

Code

实现时可以写成莫队的形式。具体来说，对于 PreSolution2 的部分，先做树链剖分，确定 dfs 序，所有点 uuu 按照 bg[u]bg[u]bg[u] 从小到大排序，然后在 dfs 序上跑莫队。它的过程恰 PreSolution2 的逆过程（如果按照 bg[u]bg[u]bg[u] 从大到小排序，这两个过程就完全一致），于是时间复杂度也一致。
